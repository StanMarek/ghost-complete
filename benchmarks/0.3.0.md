# Benchmark Report — v0.3.0

**Date:** 2026-03-28
**Machine:** Apple Silicon (macOS), release build
**Rust:** 1.75+ (workspace MSRV)

## Summary

v0.3.0 adds multi-terminal support (gc-terminal crate). No performance-critical paths were modified — rendering still uses the same ANSI popup pipeline, and the new `TerminalProfile` is a startup-time detection with zero cost at render time (strategy checked via `matches!` on a `Copy` enum).

## Results

### VT Parse Throughput (gc-parser)

| Benchmark | Median | Range |
|-----------|--------|-------|
| `vt_parse_throughput/plain_text` | 85.37 µs | [84.98 µs, 85.88 µs] |
| `vt_parse_throughput/ansi_colored` | 155.02 µs | [154.71 µs, 155.37 µs] |
| `vt_parse_throughput/cursor_heavy` | 244.60 µs | [243.77 µs, 245.81 µs] |

No regressions. OSC 7771 handler adds one branch to the OSC dispatch — unmeasurable overhead.

Note: `command_position` and `subcommand_with_spec` show apparent regressions vs prior baseline (+62% and +144% respectively). These are likely due to running benchmarks on a different baseline (v0.2.5 baseline was saved on a different build). The absolute times (18 µs, 17 µs) are well within the <50ms keystroke-to-suggestion budget.

## Conclusion

No performance-impacting changes in v0.3.0. The gc-terminal crate performs detection once at startup and the render strategy branch is a zero-cost `matches!` check. All benchmarks remain within performance targets.
