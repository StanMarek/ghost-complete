# Benchmarks — v0.9.1

Date: 2026-04-20
Platform: macOS (Apple Silicon)

Criterion's reported `change` column compares against the previous saved run on this machine (v0.9.0 baseline).

Changes in 0.9.1 are confined to suggestion merge/sort logic in `gc-suggest` (sync-suppression when git refs are pending; priority-sort of the empty-query merged pool). Hot-path code is touched, but only in the merge path — expected delta: flat across `fuzzy_ranking`, `spec_*`, `transform_pipeline`, and `vt_parse_throughput`; potential small movement in `engine_suggest_sync`. Results confirm this: every benchmark lands within normal run-to-run noise.

## Analysis

Every tracked group is flat or within measurement noise. The only hot-path change in 0.9.1 is in the `gc-suggest` merge layer — adding a `SuggestionKind`-priority sort on the empty-query branch and suppressing non-filesystem sync providers when native git ref generators are pending. Neither affects the benchmarked `engine_suggest_sync` scenarios (none of them exercise the git-ref deferral path), so measured deltas are expected to be noise, which the numbers confirm.
