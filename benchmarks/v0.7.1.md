## Summary

Patch release covering the audit remediation sweep — security hardening, crash resistance, mutex-poison recovery, tokenizer parity, async git migration, bounded script stdout, LRU cache eviction, and regex precompile at spec load.

**Trade-offs worth flagging:**

- **VT parser regressed (+4% to +100%)** — `performer.rs` now honors the `vte` ignore flag in `csi_dispatch`/`esc_dispatch` and the 200-param overflow path. Absolute numbers remain well under the <1ms PTY-forwarding budget and well below the <50ms keystroke target; these are microbench throughputs over tens of thousands of iterations, not per-keystroke latency. Correctness win (drops malformed sequences instead of dispatching them) is worth the throughput cost.
- **Spec loading +17%** — one-time startup cost from precompiling regex patterns at spec load. Pays for itself with **transform_pipeline/regex −27%** on every generator invocation that uses regex extraction. Net-positive over a session.
- **Fuzzy ranking 10k +5–8%** — likely variance; no code changes touch `nucleo` or ranking paths in this release.

**Runtime wins:**

- `engine_suggest_sync/command_position` −6.4% (async git migration reduces blocking on hot path).
- `transform_pipeline/regex` −27% (precompile).
- `transform_pipeline/json` −4% (incidental).
- `spec_resolution/deep` −4% (shallow +4% is noise-band).

No keystroke-latency regressions. All runtime paths remain inside performance targets.
